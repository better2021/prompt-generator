一个好提示词的 7 个组成部分

按这个顺序写，AI 就能准确理解你要什么。缺哪块，AI 就会自己"脑补"，脑补的往往不是你想要的。
1
任务目标 — 我要做什么
开头一句话说清楚这次要做什么。是"搭初始骨架"、"写某个功能"、还是"改已有代码"。
✅ 我要搭一个 XX 项目的初始骨架，只需要基础结构跑通，不需要完整功能
2
技术栈 — 用什么工具
前端框架、样式库、后端、数据库、ORM、认证、部署 — 每一项都点名。不说就让 AI 猜，猜错了要返工。
前端：Next.js 14 App Router + Tailwind + shadcn/ui
后端：Next.js API Routes
数据库：PostgreSQL + Prisma ORM
认证：Clerk
部署：Vercel
3
数据模型 — 核心表结构
列出主要的表和字段，说明表之间的关系（谁是谁的外键）。这是 AI 生成代码的基础，越准确越好。
User：id, email, name, role
Post：id, userId(FK→User), title, content, published
Comment：id, postId(FK→Post), userId(FK→User), body
4
输出清单 — 要哪些文件
明确告诉 AI 要生成哪些文件，用编号列出来。这样 AI 不会遗漏，也不会生成多余的东西。
请生成以下文件：
1. prisma/schema.prisma
2. lib/db.ts（Prisma 单例）
3. app/api/posts/route.ts（GET + POST）
4. .env.example
5
边界 — 不要什么
告诉 AI 本次不做什么，避免它生成一大堆你不需要的代码，反而干扰重点。
本次不需要：
- UI 页面（只要后端 API）
- 完整的邮件发送功能（留 placeholder 即可）
- 测试文件
6
代码规范 — 怎么写
说明用 TypeScript、要加注释、错误处理方式、命名风格等。一次说好，后面迭代风格统一。
代码要求：
- 全部用 TypeScript，严格类型
- API 返回统一格式：{data, error}
- 数据库操作加 try/catch
- 关键逻辑加中文注释
7
最后要求 — 怎么给我
要求 AI 按文件分开输出（不要一坨混在一起），并说明下一步怎么跑起来。
请按文件分开输出，每个文件前标明文件路径。
最后给出运行步骤（npm install、prisma migrate、npm run dev）。
